Introduction
YOLO (You Only Look Once) is a neural network that detects objects and provides the bounding
box around them. It is widely used because of its speed and accuracy.
The model must be trained on examples of the objects to be detected. So, to detect Lego’s and
or RoboMaster’s as needed for Projects 2 and 3 you will need to collect a small dataset
(probably 100 images), label it, and fine-tune the model (train it).

Step 1: Create a Labeled Dataset
Refer to the sample dataset for a quick overview. Detailed instructions are provided below:
Note: We encourage teams to share their datasets with each other.
General Instructions:
 Capture a sequence of about 100 images of the object to be detected
o Use the RoboMaster’s camera
 If you use your cell phone to capture the dataset, the model may still
work, but it is best to use the same camera for training and testing
o Be sure to vary the pose, lighting conditions, and background as possible
o Make sure there are several images without any objects in them
 Otherwise, the algorithm will learn to always detect an object
 Split the images into train and validation sets
o Put about 80% into a folder named “train”
o Put the remaining in a folder named “validation”
o Create a file called “names.txt”. Put the name of each object on a separate row.
 Label the dataset
o There are many tools available to create a labeled dataset for YOLO. We currently
recommend Yolo Label.
 Download and install Yolo Label following these instructions
o Label the dataset by drawing bounding boxes
 Refer to the Yolo Label README for detailed usage instructions and
keyboard shortcuts
o Images without any objects to be detected in them need to be labeled as well
 Just viewing them in Yolo Label and then skipping to the next image will
create this “empty” label.
 Create the dataset folder structure
o Create a folder named “cmsc477_yolo”
 Inside, create a folder named “datasets”
 Put the “train” and “validation” folders in “datasets”
 Create a “data.yaml” file that configures the “datasets”, “train”, and “validation” directory
and lists the objects in the dataset:
o For example here is the “data.yaml” for our sample dataset:.
path: ./ # After first run, YOLO expects all datasets to be in path+"/datsets"
train: robomaster_train
val: robomaster_validation
nc: 1 # Number of classes (1 for only "object")
names: ["robomaster"]

Step 2: Fine-Tune (train) the Model
From the “cmsc477_yolo” folder run:
“yolo detect train data=data.yaml model=v8s.pt imgsz=640 epochs=50”
This command will create a folder called “runs/detect/train{N}” where N is the number of times
the command has been run. Keep track of this so that the testing below is run on the correct
model parameters.
The flag “model=yolov8s.pt” means a pre-trained YOLOv8s model will be used as a starting
point. The training procedure fine-tunes that model for our specific objects. This allows training
to take much less time, and accuracy improves because the network was pre-trained on a
diverse set of objects.

Model Selection: In the TAs testing, YOLOv8s appears to be more accurate than YOLOv11s for

detecting the robomaster and sometimes misses it.

Bug Workaround: Yolo memorizes the location of the “datasets” folder and expects it to remain
in the same place forever. If you need to reset this, run “yolo settings” and delete the settings
file it prints the location of. Bug report is here.
Programmatic Training: It is possible to write a Python program that does the training, see here.
This may be helpful if you train on Google Colab (where a GPU is available for free).

Step 3: Test the Model
To run live on the robomaster:
Use the code provided in Project 2 Description’s Appendix. Just modify the model path to point
to your new weights.
To run on a recorded video:
From the “cmsc477_yolo” folder run:
yolo detect predict model=runs/detect/train{N_train}/weights/best.pt source=test_video.mp4
The results will be in “runs/detect/predict{N_predict}”
In the provided sample, the video is in “runs/detect/predict/test_video.avi”
Performance: GPU: 5-6ms per frame, CPU: 50-60ms per frame
To run a folder of images:
From the “cmsc477_yolo” folder run:
yolo detect predict model=runs/detect/train{N_train}/weights/best.pt
source=datasets/robomaster_validation

The results will be in “runs/detect/predict{N_predict}”
